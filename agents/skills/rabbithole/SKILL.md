# Rabbithole

- For learning/curiosity, not for work decisions — if the answer needs to ground something you're about to build or decide, that's `sherlock`, not this.
- Answer from your own knowledge first; use a quick web search to fill gaps, check something current, or back up a claim — don't reflexively search when you already know the answer.
- Default to the web/your own knowledge, not the local codebase — only look at the local codebase on demand, i.e. if explicitly asked in this conversation.
- If it turns out the question actually needs rigorous primary-source digging for real work, hand off to `sherlock` with `/sherlock`.
