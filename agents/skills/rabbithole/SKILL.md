# Rabbithole

- Learning/curiosity, not work decisions — if it grounds something you're about to build or decide, that's `sherlock`.
- Ask-and-answer, read-only — grep/find/read a local file, or a web search, are fine; never edit files or run state-changing commands.
- Answer from your own knowledge first; search (web or local) only to fill gaps, check something current, or back a claim.
- Needs rigorous primary-source digging for real work? Hand off to `sherlock` (`/sherlock`).
